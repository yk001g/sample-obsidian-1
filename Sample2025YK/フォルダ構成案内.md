# 📁 フォルダ・ファイル構成案内

このリポジトリは、**汎用セカンドブレイン**として設計されたObsidian知識管理システムです。

---

## 🗂️ クイックビュー

```
second-brain-vault/
├─ 00_Memo/          # 🎤 何でも放り込む（5秒でメモ）
├─ 01_Inbox/         # 📥 Cursor処理済み（週1回整理）
├─ 02_Daily/         # 📅 毎日の記録（エピソード記憶）
├─ 03_Input/         # 🔥 今週〜今月使う資料
├─ 04_Memory/        # 🧠 永久保存の知識（MOC管理）
├─ 05_Output/        # 🎯 プロジェクト・継続活動
│  ├─ Projects/      #   期限あり（@Active/@Planning/@Completed）
│  └─ Areas/         #   継続的（@TODO/@Doing/@Completed）
├─ 06_Templates/     # 📋 テンプレート集
├─ 07_System/        # ⚙️ システム設定・ダッシュボード
├─ 08_prompts/       # 📝 プロンプト集
├─ 99_Archive/       # 🗄️ アーカイブ（過去データ）
├─ AGENTS.md         # 📖 詳細ルール（必読）
└─ README.md         # 📄 基本説明
```

---

## 📋 各フォルダの役割

### 00_Memo/ - 何でも放り込む場所
**役割**: 5秒以内にメモできる、摩擦ゼロの場所

- ✅ 何でも放り込む（ファイル名適当でOK）
- ✅ 整理不要（週1回01_Inboxへ移動）
- ⚠️ 最長1週間で処理

**例**: `memo-001.md`, `アイデア.md`, `voice-20250113.m4a`

---

### 01_Inbox/ - Cursor処理済み
**役割**: 00_Memoから整理されたmdファイルが入る

- ✅ タグ・双方向リンク・YAML frontmatter付き
- ✅ ファイル名がわかりやすい
- ⚠️ 1週間以内に振り分け（目標: 10ファイル以下）

**振り分け先**:
- `03_Input` (今週使う)
- `04_Memory` (3ヶ月後も見る)
- `05_Output` (プロジェクト/エリア関連)
- `02_Daily` (日次記録)

---

### 02_Daily/ - 毎日の記録
**役割**: エピソード記憶（いつ、どこで、何をしたか）

**構造**:
```
02_Daily/
├─ 2025/
│  ├─ 2025-01/
│  │  ├─ 2025-01-13/
│  │  │  ├─ 2025-01-13-Daily.md   ← 必須
│  │  │  ├─ 2025-01-13-TODO.md    ← 必須
│  │  │  └─ [任意ファイル].md     ← 会議メモ、学習メモなど
│  │  └─ ...
│  └─ ...
├─ Weekly-Reviews/2025/
└─ Monthly-Reviews/2025/
```

---

### 03_Input/ - 短期記憶
**役割**: 今まさに使っている情報の作業台

**置くもの**:
- 今週のプロジェクトで参照する資料
- 今月学んでいるトピックのメモ
- 頻繁に見るチートシート

**クリーンアップ**: 毎週日曜にチェック（1週間触ってない→移動）

---

### 04_Memory/ - 長期記憶
**役割**: 3ヶ月後、1年後、5年後も価値がある知識の保管庫

**構造**:
```
04_Memory/
├─ AI/
│  ├─ _AI-MOC.md          ← マスターマップ
│  ├─ Tools/
│  ├─ Techniques/
│  └─ Concepts/
├─ Education/
├─ Business/
├─ Technical/
├─ Personal/
└─ _Master-Index.md      ← 全体マップ
```

**MOC（Map of Contents）**: 各カテゴリーの知識地図

---

### 05_Output/ - アウトプット管理

#### Projects/ - 期限ありプロジェクト
```
Projects/
├─ @Active/         # 進行中
├─ @Planning/       # 企画中
└─ @Completed/     # 完了
```

**例**: YouTube動画シリーズ、SURVIBE AI講座、企業研修

#### Areas/ - 継続的エリア（期限なし）
```
Areas/
├─ Content-Creation/YouTube-Channel/
│  ├─ @TODO/        # アイデア段階
│  ├─ @Doing/       # 制作中
│  └─ @Completed/   # 公開済み
├─ Business/
├─ Personal/
└─ Community/
```

**例**: YouTubeチャンネル運営、企業研修事業

---

### 06_Templates/ - テンプレート集
**役割**: よく使うフォーマットを保存

- Daily/ - デイリーノート、TODO、レビュー
- Content/ - YouTube、ブログ、SNS
- Projects/ - プロジェクト管理
- Knowledge/ - 知識管理
- Meeting/ - 会議ノート

---

### 07_System/ - システム管理
**役割**: Obsidian自体の設定・ワークフロー

- Dashboards/ - 🏠-HOME.md、📊-Weekly-Dashboard.md など
- Workflows/ - 日次・週次ワークフロー
- Scripts/ - 自動化スクリプト
- Settings/ - プラグイン、設定
- Documentation/ - システムドキュメント

---

### 08_prompts/ - プロンプト集
**役割**: 効果的だったプロンプトをカテゴリ別に保存

- 01_整理系/ - Inbox整理、Memory作成
- 02_生成系/ - Daily作成、コンテンツ生成
- 03_レビュー系/ - 品質チェック、レビュー
- 04_学習系/ - 技術調査、知識収集
- 05_改善系/ - コマンド改善、システムメンテナンス
- 06_一括作業系/ - 全量作業、一括処理

---

### 99_Archive/ - アーカイブ
**役割**: 役目を終えたファイルの墓場（でも完全には消さない）

- 年別に整理（2024/, 2023/ など）
- 検索はできる状態を保つ

---

## 🔄 情報の流れ

```
00_Memo (生まれた瞬間)
   ↓ Cursorで処理
01_Inbox (整理済み)
   ↓ 週1回振り分け
03_Input (今週使う)
   or
04_Memory (ずっと使う)
   or
05_Output (アウトプット)
   ↓
99_Archive (役目を終えた)
```

## 📝 命名規則

### ファイル名（日本語）
- ✅ `Cursor-キーボードショートカット.md`
- ✅ `2025-01-13-クライアントABCとの打ち合わせ.md`
- ❌ `memo.md`（抽象的すぎ）
- ❌ `新しい ファイル.md`（スペース禁止）

### フォルダ名（英語）
- ✅ `04_Memory/AI/Tools/Cursor/`
- ✅ `05_Output/Projects/@Active/`
- ✅ `02_Daily/2025/2025-01/`

**重要**: **フォルダ名は英語、ファイル名は日本語**
