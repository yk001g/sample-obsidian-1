# 📥 Inbox処理フロー

**実行タイミング**: 週1回（日曜夜推奨）

---

## 🎯 目的

01_Inbox内のファイルを適切なフォルダに振り分ける。

---

## 📋 処理フロー

### Step 1: Inboxファイルの確認
```bash
# 01_Inbox内の全ファイルを確認
```

**実行内容:**
- [ ] 01_Inbox内のファイル数をカウント
- [ ] 各ファイルの内容を確認
- [ ] **目標**: 10ファイル以下

---

### Step 2: 振り分け判定

#### 判定フロー

```
このファイル、どこに行くべき？

┌─ プロジェクト関連？
│   YES → 05_Output/Projects/@Active/[Project]/
│   NO ↓
│
┌─ 継続エリア関連？
│   YES → 05_Output/Areas/[Area]/@TODO/
│   NO ↓
│
┌─ 汎用知識？
│   YES → 04_Memory/[Category]/
│   NO ↓
│
┌─ 今週使う資料？
│   YES → 03_Input/
│   NO ↓
│
┌─ デイリーログ？
│   YES → 02_Daily/YYYY/YYYY-MM/YYYY-MM-DD/
│   NO ↓
│
└─ よくわからない
    → とりあえず03_Input
    （来週また考える）
```

---

### Step 3: プロジェクト関連の判定

**判定基準:**
- 具体的なプロジェクトについて書かれている
- プロジェクトの計画・設計・実装関連
- プロジェクト固有の内容

**移動先:**
- `05_Output/Projects/@Active/[Project]/`
- または `05_Output/Projects/@Planning/[Project]/`

**例:**
- SURVIBE AI 12月講座の企画書
- YouTube動画シリーズの計画
- 企業研修プロジェクトの仕様書

---

### Step 4: 継続エリア関連の判定

**判定基準:**
- 継続的な活動（期限なし）
- 長期的なエリア管理
- 継続的なコンテンツ作成

**移動先:**
- `05_Output/Areas/[Area]/@TODO/`
- または `05_Output/Areas/[Area]/@Doing/`

**例:**
- YouTubeチャンネル運営（継続的）
- Survibe AI Baib-Coding-School（継続的）
- 企業研修事業（継続的）

---

### Step 5: 汎用知識の判定

**判定基準:**
- 3ヶ月後も参照する価値がある
- 自分の言葉で書かれている
- 完結した内容

**移動先:**
- `04_Memory/[Category]/[Subcategory]/`

**カテゴリ:**
- `AI/` - AI関連の知識
- `Business/` - ビジネス関連の知識
- `Education/` - 教育関連の知識
- `Personal/` - 個人関連の知識
- `Technical/` - 技術関連の知識

---

### Step 6: 今週使う資料の判定

**判定基準:**
- 今週〜今月使う資料
- 頻繁に見るチートシート
- 進行中のタスクリスト

**移動先:**
- `03_Input/`

**例:**
- 今週のプロジェクトで参照する資料
- 今月学んでいるトピックのメモ
- 今週の焦点（this-week-focus.md）

---

### Step 7: デイリーログの判定

**判定基準:**
- 日次記録
- その日の出来事・気づき
- 会議メモ

**移動先:**
- `02_Daily/YYYY/YYYY-MM/YYYY-MM-DD/`

**例:**
- 2025-01-13-Daily.md
- 2025-01-13-TODO.md
- meeting-client-abc.md

---

## 🔧 自動化

### Cursorコマンド
```bash
/organize-inbox
```

**実行内容:**
- 01_Inbox内の全ファイルを自動振り分け
- 内容を分析して適切なフォルダへ移動
- 判定理由を記録
