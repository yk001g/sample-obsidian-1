# フォルダ構造ガイド

**バージョン**: 1.0  
**最終更新**: 2025-01-13

---

## 📁 フォルダ構成の全体像

```
SecondBrain_Vault/
├─ 00_Memo/          # 何も考えず放り込む
├─ 01_Inbox/         # Cursor処理済み（週1回整理）
├─ 02_Daily/         # 毎日自動生成
├─ 03_Input/         # 今週〜今月使う資料
├─ 04_Memory/        # 永久保存の知識
├─ 05_Output/        # プロジェクト・継続活動
├─ 06_Templates/     # テンプレート集
├─ 07_System/        # システム設定・ダッシュボード
├─ 08_prompts/       # プロンプト集
└─ 99_Archive/       # 終わったもの
```

---

## 📂 各フォルダの詳細

### 00_Memo - 何でも放り込む場所

#### 役割
**5秒以内にメモできる、摩擦ゼロの場所**

#### ルール
- ✅ 何でも放り込む
- ✅ ファイル名適当でOK
- ✅ 整理しなくてOK
- ❌ 整理しようとする（時間の無駄）
- ❌ フォルダ分けする（意味がない）

#### ファイルの寿命
- **最長1週間**: 1週間以上放置されたら、週次レビューで処理
- **理想は3日以内**: 新鮮なうちに01_Inboxへ

---

### 01_Inbox - Cursor処理済みの場所

#### 役割
**00_Memoから取り出して、Cursorで「Obsidian用に整理されたmdファイル」が入る場所**

#### ルール
- ✅ タグ付き
- ✅ 双方向リンク付き
- ✅ YAML frontmatter付き
- ✅ ファイル名がわかりやすい
- ❌ フォルダ分けする
- ❌ 長期保存する（1週間が限度）

#### 処理タイミング
- **毎週日曜夜**: 01_Inboxを空にする
- **目標**: 10ファイル以下を保つ

---

### 02_Daily - 毎日の記録

#### 役割
**エピソード記憶（いつ、どこで、何をしたか）を時系列で保存**

#### フォルダ構造
```
02_Daily/
├─ 2025/
│  ├─ 2025-01/
│  │  ├─ 2025-01-13/
│  │  │  ├─ 2025-01-13-Daily.md
│  │  │  ├─ 2025-01-13-TODO.md
│  │  │  └─ meeting-client-abc.md
│  │  └─ ...
│  └─ ...
├─ Weekly-Reviews/
└─ Monthly-Reviews/
```

#### 命名規則
- 必須ファイル: `YYYY-MM-DD-Daily.md`, `YYYY-MM-DD-TODO.md`
- 任意ファイル: `meeting-[相手]-[トピック].md`, `learning-[トピック].md`

---

### 03_Input - 短期記憶（今週〜今月）

#### 役割
**今まさに使っている情報の作業台**

#### 使い方
- ✅ 今週のプロジェクトで参照する資料
- ✅ 今月学んでいるトピックのメモ
- ✅ 頻繁に見るチートシート
- ❌ 1ヶ月以上使っていない資料

#### クリーンアップルール
- **毎週日曜にチェック**
- 1週間触ってないファイル → 04_Memory or 99_Archive

---

### 04_Memory - 長期記憶（永久保存）

#### 役割
**3ヶ月後、1年後、5年後も価値がある知識の保管庫**

#### 構造
```
04_Memory/
├─ AI/
│  ├─ _AI-MOC.md
│  ├─ Tools/
│  ├─ Techniques/
│  └─ Concepts/
├─ Business/
├─ Education/
├─ Personal/
├─ Technical/
└─ _Master-Index.md
```

#### 3ヶ月ルール
**質問**: 3ヶ月後も参照する価値がある？
- YES → 04_Memory
- NO → 02_Daily or 99_Archive

---

### 05_Output - アウトプット管理

#### 役割
**プロジェクト（期限あり）と継続エリア（期限なし）を管理**

#### 構造
```
05_Output/
├─ Projects/           # 期限ありプロジェクト
│  ├─ @Active/         # 進行中
│  ├─ @Planning/       # 企画中
│  └─ @Completed/      # 完了
└─ Areas/              # 継続的エリア
   ├─ Content-Creation/
   ├─ Business/
   ├─ Personal/
   └─ Community/
```

#### ProjectsとAreasの違い
| 項目 | Projects | Areas |
|------|----------|-------|
| 期限 | ✅ ある | ❌ ない |
| 終わり | ✅ ある | ❌ ない |
| 例 | YouTube動画シリーズ | YouTubeチャンネル運営 |

---

### 06_Templates - テンプレート集

#### 役割
**よく使うフォーマットを保存して、コピペの手間を削減**

#### 構造
```
06_Templates/
├─ Daily/
├─ Content/
├─ Projects/
├─ Knowledge/
└─ Meeting/
```

---

### 07_System - システム管理

#### 役割
**Obsidian自体の設定、ダッシュボード、ワークフローを管理**

#### 構造
```
07_System/
├─ Dashboards/          # ダッシュボード
├─ Workflows/           # ワークフロー
├─ Scripts/             # 自動化スクリプト
├─ Settings/            # 設定
└─ Documentation/       # ドキュメント
```

---

### 08_prompts - プロンプト集

#### 役割
**効果的だったプロンプトをカテゴリ別に保存**

#### 構造
```
08_prompts/
├─ 01_整理系/
├─ 02_生成系/
├─ 03_レビュー系/
├─ 04_学習系/
├─ 05_改善系/
└─ 06_一括作業系/
```

---

### 99_Archive - アーカイブ

#### 役割
**役目を終えたファイルの墓場（でも完全には消さない）**

#### 構造
```
99_Archive/
├─ 2024/
├─ 2023/
└─ _archive-workflow.md
```

## 🔄 フォルダ間の使い分けガイド

### Memory vs Input
- **04_Memory**: 3ヶ月後も参照する価値がある知識（長期記憶）
- **03_Input**: 今週〜今月使う資料（短期記憶）

### Projects vs Areas
- **05_Output/Projects**: 期限ありのプロジェクト
- **05_Output/Areas**: 継続的な活動（期限なし）

### Daily vs Memory
- **02_Daily**: その日の記録（エピソード記憶）
- **04_Memory**: 長期記憶として残す知識
