# Note 戦略・運用ガイド

## 概要
noteプラットフォームでのコンテンツ配信戦略。深い考察記事を通じて、AI・テクノロジー・教育分野での思想リーダーシップを確立する。

---

## 🎯 目的・ゴール

### 主要目的
1. **思考の整理と言語化**
   - 日々の気づきや洞察を体系的に記事化
   - 複雑な概念を噛み砕いて説明

2. **思想リーダーシップの確立**
   - AI/テクノロジー分野での独自の視点
   - 現場感のある深い考察
   - 実践的な示唆の提供

3. **コミュニティとの対話**
   - 読者からのフィードバック
   - 議論の喚起
   - 信頼関係の構築

4. **他メディアへの展開**
   - YouTube、X、ブログへの素材
   - セミナー・講座のコンテンツベース
   - クライアント提案の根拠

---

## 📝 コンテンツ方針

### 記事のテーマ（3本柱）

#### 1. AI・テクノロジー考察
```
主なトピック:
- AIツールの深層分析（Cursor, Claude, Dify等）
- AI開発の未来予測
- テクノロジートレンドの構造的理解
- インフラ・エコシステムの変化

特徴:
- 表面的なレビューではなく構造分析
- 「なぜそうなるか」の深掘り
- 多角的な視点（技術/ビジネス/人間）
```

#### 2. 教育・学習方法論
```
主なトピック:
- AI時代の学び方
- 効果的なティーチング技法
- カリキュラム設計の考え方
- 受講生の成長プロセス

特徴:
- 現場での実践ベース
- 具体的な事例
- 再現可能な方法論
```

#### 3. 働き方・生き方
```
主なトピック:
- エンジニアのキャリア戦略
- 生産性とクリエイティビティ
- 家族とのバランス
- 個人の価値創造

特徴:
- 等身大の語り
- 成功だけでなく葛藤も
- 具体的なアクション
```

## 🎨 文章スタイル

### スタイルの特徴
詳細は [[文章スタイルガイド]] を参照

**基本方針**:
- カジュアルと知的のバランス
- 友人と深夜に語り合うような親密さ
- 上から目線にならない
- でも媚びない

**頻出パターン**:
```
「〜じゃないですか」  → 共感形成
「〜なんだよな」      → 実感を込めた結論
「〜ってこと」        → 因果関係の説明
「で、面白いのが〜」  → 転換・深掘り
「さあ、〜」          → 前向きな締め
```

## 📅 運用ルール

### 投稿頻度
```
目標: 週1〜2本
最低: 週1本
理想: 毎週月曜・木曜

柔軟性:
- 質を優先、無理に書かない
- ネタがあれば週3本でもOK
- 長期記事は2週に1本でもOK
```

### ワークフロー

#### 1. アイデア段階（@TODO）
```
場所: @TODO/

やること:
□ タイトル案を書く
□ 伝えたいメッセージを一行で
□ 主要な論点を3〜5個リストアップ
□ 参考資料をリンク

判断基準:
- 自分が本当に書きたいか
- 読者に価値があるか
- 今書くべきタイミングか
```

#### 2. 執筆中（@Doing）
```
場所: @Doing/

やること:
□ アウトライン作成
□ 各セクションの下書き
□ 比喩・表現の精錬
□ 読み返して推敲
□ 文章スタイルガイドでチェック

目安時間:
- 短文（2000字）: 2〜3時間
- 中文（4000字）: 4〜6時間
- 長文（6000字+）: 1〜2日
```

#### 3. 公開後（@Completed）
```
場所: @Completed/YYYY/

やること:
□ 公開URLを記録
□ 初期反応をメモ
□ 数日後の数値を記録
□ フィードバックを整理
□ 改善点を次に活かす

記録項目:
- 公開日時
- ビュー数
- スキ数
- コメント（重要なもの）
- 学び・気づき
```

## 📊 評価指標

### 定量指標
```
重視する:
- ビュー数（どれだけ読まれたか）
- スキ数（どれだけ響いたか）
- コメント数（どれだけ議論を喚起したか）

参考程度:
- フォロワー増加数
- SNSでのシェア数
```

### 定性指標
```
最重要:
- 自分が納得できる記事か
- 本当に伝えたいことを伝えられたか
- 読者から建設的なフィードバックがあったか

重要:
- 他メディアへの展開可能性
- クライアントからの反応
- 自分の思考が深まったか
```

## 🔄 他メディアとの連携

### noteを起点とした展開

```
note記事（深い考察）
  ↓
├─ X: 要点を短文で拡散
├─ YouTube: 動画化（より詳しく）
├─ ブログ: 技術記事として再構成
├─ セミナー: 講義コンテンツに
└─ LinkedIn: ビジネス向けに編集
```

### 逆方向の流れ
```
YouTube動画
  ↓
note記事: 文字起こし + 深掘り考察

X投稿（複数）
  ↓
note記事: まとめて体系化

セミナー講義
  ↓
note記事: 内容の言語化と共有
```

## 💡 記事アイデアの源泉

### 日常からの抽出
```
✅ 良い記事ネタ:
- 「最近モヤモヤしてること」
- 「クライアントとの会話で気づいたこと」
- 「ツールを使ってて感じた違和感」
- 「受講生の質問で考えさせられたこと」
- 「業界ニュースへの違和感や共感」

❌ 避けるべき:
- 他人の記事の焼き直し
- 表面的なHow-to（他でいい）
- 「書かなきゃ」という義務感
```

### 記録の習慣
```
□ Daily Noteに気づきをメモ
□ 00_Memoに思いついたら即メモ
□ 週次レビューでネタを振り返り
□ 月次で記事テーマを計画
```

## 🎯 成功パターン

### 過去の高評価記事の共通点
```
1. 個人的な実感から始まる
   「正直に言うと、最近...」
   
2. 表面的でなく構造を語る
   「なぜそうなるのか」の深掘り
   
3. 印象的な比喩を使う
   「建築物じゃなくて森なんだよ」
   
4. 多角的な視点
   技術 × ビジネス × 人間
   
5. 前向きな締め
   「さあ、〜」で行動を促す
```

---

## ⚠️ 避けるべきこと

### コンテンツ面
```
❌ 炎上狙いの極端な主張
❌ 根拠のない批判
❌ 他者を見下す表現
❌ バズ狙いの釣りタイトル
❌ 表面的なまとめ記事
```

### 運用面
```
❌ 無理に週1本書こうとする（質が下がる）
❌ 数字だけを追う（本質を見失う）
❌ 他人と比較する（自分の軸がブレる）
❌ 完璧主義で公開しない（機会損失）
```
