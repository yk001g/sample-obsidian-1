# Inboxレビュープロンプト

## 使用タイミング
週1回（日曜夜）にInboxを分析して整理案を提示する時

## プロンプト本文
Inboxを分析して整理案を提示してください。

実行内容:
1. `01_Inbox/`内の全ファイルをスキャン
2. 各ノートを以下の基準で評価:
   - Memory候補 (7点以上、3ヶ月後も価値あり)
   - Input候補 (今週〜今月使う)
   - Output候補 (プロジェクト・継続活動)
   - Daily候補 (日次記録)
   - 要追加情報 (4-6点)
   - 削除候補 (3点以下)

評価基準 (10点満点):
- 自分の言葉で書かれているか: 3点
- 1ノート1アイデアか: 2点
- 完結性: 2点
- リンクがあるか: 2点
- タグがあるか: 1点

振り分け先の判定:
- Memory候補（7点以上、3ヶ月後も価値あり）→ `04_Memory/`
- Input候補（今週〜今月使う）→ `03_Input/`
- Output候補（プロジェクト・継続活動）→ `05_Output/`
- Daily候補（日次記録）→ `02_Daily/`
- Archive候補（判断に迷う）→ `99_Archive/`

## 使用例
```
/inbox-review
```

## 期待される結果
- Inbox整理レポートが生成される
- 各ファイルのスコアと振り分け先が表示される
- アクション提案が表示される

## 注意事項
- 評価基準を厳密に適用
- 振り分け先の判定理由を明確に
