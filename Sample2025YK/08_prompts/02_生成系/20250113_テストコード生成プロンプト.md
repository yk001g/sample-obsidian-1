# テストコード生成プロンプト

## 使用タイミング
JestとPlaywrightのテストコードを自動生成したい時

## プロンプト本文
```
このプロジェクトを全体的に網羅して解析して、jestとplaywrightのテスト設計mdを境界値と分岐と例外処理の正常系異常系など省略せずに１つずつ丁寧にケースを書いて、npm run test(jest)とnpm run test:e2e(playwright)が全てのテスト合格するまでコマンド実行とテストコード修正を繰り返して、データベース接続部分はmock作ってテストコード実装して。

全APIのjest全テストケースと全画面の全分岐のplayrightテストケースをそれぞれ詳細にdocs/testフォルダにmdファイルで別々のmdファイルに設計書いて、
その設計通りにテストソース書いて実施して
```

## 使用例
プロジェクト全体のテストコードを自動生成する場合

## 期待される結果
1. プロジェクトが全体的に網羅して解析される
2. JestとPlaywrightのテスト設計mdが作成される
   - 境界値と分岐と例外処理の正常系異常系など省略せずに1つずつ丁寧にケースが書かれる
3. `npm run test`(jest)と`npm run test:e2e`(playwright)が全てのテスト合格するまでコマンド実行とテストコード修正が繰り返される
4. データベース接続部分はmockが作られてテストコード実装される
5. 全APIのjest全テストケースと全画面の全分岐のplaywrightテストケースがそれぞれ詳細に`docs/test`フォルダにmdファイルで別々のmdファイルに設計される
6. その設計通りにテストソースが書かれて実施される

## 注意事項
- テスト設計のベストプラクティスを参照
- 境界値と分岐と例外処理を省略しない
- データベース接続部分は必ずmockを作る
