# Obsidianのよくある問題と解決策

## 概要
Obsidianを使用する際によく遭遇する問題と、その解決策をまとめたトラブルシューティングガイドです。初心者から上級者まで、実践的な解決方法を提供します。

## 詳細

### 問題カテゴリー

1. **インストール・セットアップ関連**
2. **パフォーマンス関連**
3. **リンク・検索関連**
4. **プラグイン関連**
5. **データ管理関連**

## 基本的な使い方

### 問題1: Vaultが開けない

#### 症状
- Vaultを開こうとするとエラーが表示される
- Vaultのパスが見つからない

#### 解決策
1. **Vaultのパスを確認**
   - 設定 → ファイルとリンク → Vaultの場所を確認
   - フォルダが存在するか確認

2. **ファイルのアクセス権限を確認**
   - フォルダの読み書き権限があるか確認
   - 管理者権限が必要な場合がある

3. **Obsidianを再起動**
   - Obsidianを完全に終了して再起動
   - キャッシュの問題が解決される場合がある

### 問題2: ノートが保存されない

#### 症状
- ノートを編集しても保存されない
- 変更が失われる

#### 解決策
1. **自動保存の設定を確認**
   - 設定 → ファイルとリンク → 自動保存の間隔を確認
   - 手動保存: `Ctrl+S`（Windows/Linux）または`Cmd+S`（Mac）

2. **ファイルの権限を確認**
   - フォルダの書き込み権限があるか確認
   - 読み取り専用になっていないか確認

3. **ディスク容量を確認**
   - ディスクに空き容量があるか確認

## 高度な使い方

### 問題3: パフォーマンスが遅い

#### 症状
- ノートの読み込みが遅い
- 検索が遅い
- グラフビューが重い

#### 解決策

##### 1. プラグインを見直す
```
- 使っていないプラグインを無効化
- 重いプラグインを特定して削除
- プラグインの設定を最適化
```

##### 2. ノート数を減らす
```
- 古いノートをアーカイブ
- 不要なノートを削除
- 添付ファイルを外部ストレージに移動
```

##### 3. インデックスの再構築
```
- 設定 → ファイルとリンク → インデックスの再構築
- キャッシュのクリア
```

##### 4. 検索範囲を限定
```
- フォルダを指定して検索
- Dataviewクエリの範囲を限定
```

### 問題4: リンクが切れている

#### 症状
- リンクをクリックしてもノートが開かない
- リンクが点線で表示される（未作成ノート）

#### 解決策

##### 1. ノート名の確認
```
- リンク先のノートが存在するか確認
- ノート名のスペルミスを確認
- 大文字小文字を確認
```

##### 2. リンク名の変更
```
- Obsidianの「リンク名変更」機能を使用
- 自動でリンクが更新される
- 手動でリンクを修正
```

##### 3. リンク切れチェックプラグイン
```
- "Broken Link Highlighter"プラグインをインストール
- リンク切れを自動検出
```

### 問題5: プラグインが動作しない

#### 症状
- プラグインをインストールしたが動作しない
- プラグインの設定が反映されない

#### 解決策

##### 1. プラグインの有効化確認
```
- 設定 → インストール済みプラグイン
- プラグインが有効化されているか確認
```

##### 2. プラグインのバージョン確認
```
- Obsidianのバージョンと互換性があるか確認
- プラグインを最新版に更新
```

##### 3. プラグインの競合確認
```
- 他のプラグインと競合していないか確認
- プラグインを1つずつ有効化してテスト
```

##### 4. Obsidianの再起動
```
- Obsidianを完全に終了して再起動
- プラグインの読み込みが正常に行われる
```

### 問題6: 検索結果が表示されない

#### 症状
- 検索を実行しても結果が表示されない
- 検索が動作しない

#### 解決策

##### 1. 検索範囲の確認
```
- 検索範囲が正しく設定されているか確認
- フォルダを指定している場合は範囲を確認
```

##### 2. 検索演算子の確認
```
- 検索演算子の構文を確認
- スペルミスがないか確認
```

##### 3. インデックスの再構築
```
- 設定 → ファイルとリンク → インデックスの再構築
- 検索インデックスを再構築
```

### 問題7: データが失われた

#### 症状
- ノートが消えた
- 変更が失われた

#### 解決策

##### 1. ファイル回復機能を使用
```
- 設定 → ファイルとリンク → ファイル回復
- 自動バックアップから復元
```

##### 2. バックアップから復元
```
- Gitバックアップから復元
- クラウドストレージのバージョン履歴から復元
```

##### 3. 一時ファイルを確認
```
- .obsidian/trash/フォルダを確認
- 削除されたファイルが残っている場合がある
```

## 使用例

### シナリオ1: パフォーマンス問題の解決
```
問題: Obsidianが重い
解決策:
1. プラグインを5個から2個に減らす
2. 古いノートをアーカイブ
3. インデックスを再構築

結果: パフォーマンスが改善
```

### シナリオ2: リンク切れの修正
```
問題: リンクが切れている
解決策:
1. リンク切れチェックプラグインをインストール
2. 切れているリンクを特定
3. ノート名を修正またはリンクを更新

結果: すべてのリンクが正常に動作
```

## よくある問題と解決策

### 問題8: モバイルで同期できない

#### 症状
- デスクトップとモバイルでノートが同期されない
- モバイルでノートが表示されない

#### 解決策

##### 1. Obsidian Syncの確認
```
- Obsidian Sync（有料）を使用している場合、同期設定を確認
- 同期が有効になっているか確認
```

##### 2. クラウドストレージの確認
```
- Dropbox、iCloud、Google Driveなどの同期設定を確認
- フォルダが正しく同期されているか確認
```

##### 3. 手動同期
```
- モバイルアプリで手動同期を実行
- デスクトップで変更を保存してから同期
```

### 問題9: テンプレートが挿入されない

#### 症状
- テンプレートを選択しても挿入されない
- テンプレートの変数が置換されない

#### 解決策

##### 1. テンプレートフォルダの確認
```
- 設定 → コアプラグイン → Templates
- テンプレートフォルダが正しく設定されているか確認
```

##### 2. テンプレートファイルの確認
```
- テンプレートファイルが正しいフォルダにあるか確認
- ファイル名が正しいか確認
```

##### 3. 変数の構文確認
```
- 変数の構文が正しいか確認（{{date}}など）
- Templaterプラグインを使用している場合、構文を確認（<% tp.date.now() %>）
```

4. **リンクのメンテナンス**: 定期的にリンク切れをチェック
