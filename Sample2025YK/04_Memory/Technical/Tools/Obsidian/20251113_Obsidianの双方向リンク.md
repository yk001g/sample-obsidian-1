# Obsidianの双方向リンク

## 概要
双方向リンクは、Obsidianの最も重要な機能の一つです。ノートAからノートBにリンクすると、自動的にノートBからノートAへの逆方向リンク（バックリンク）が作成され、知識のネットワーク化を実現します。

## 詳細

### 双方向リンクとは
従来のハイパーリンク（一方向）とは異なり、Obsidianのリンクは**双方向**です。

```
従来のリンク（一方向）:
ノートA → ノートB
（ノートBからノートAへのリンクは手動で作成する必要がある）

Obsidianの双方向リンク:
ノートA ⇄ ノートB
（自動的に双方向のリンクが作成される）
```

### リンクの作成方法

#### 基本的なリンク
```markdown
[[ノート名]]
```

#### エイリアス（表示名の変更）
```markdown
[[ノート名|表示名]]
例: [[20251113_Obsidianとは何か|Obsidianの基本]]
```

#### 見出しへのリンク
```markdown
[[ノート名#見出し名]]
例: [[20251113_Obsidianとは何か#概要]]
```

#### ブロックへのリンク
```markdown
[[ノート名#^block-id]]
例: [[20251113_Obsidianとは何か#^abc123]]
```

## 基本的な使い方

### リンクの作成手順
1. `[[` を入力
2. ノート名を入力（自動補完が表示される）
3. `]]` で閉じる

### リンクの編集
- **リンク名の変更**: リンクをクリック → 名前を変更
- **リンクの削除**: リンクを選択 → Deleteキー
- **リンク先の変更**: リンクをクリック → 別のノートを選択

### バックリンクの確認
1. ノートを開く
2. 右サイドバーの「バックリンク」パネルを開く
3. このノートを参照しているノートが一覧表示される

## 高度な使い方

### リンクの自動補完
```markdown
[[obs と入力すると...
→ Obsidian
→ Obsidianの双方向リンク
→ Obsidianのグラフビュー
などが候補として表示される
```

### 未作成ノートへのリンク
```markdown
[[新しいノート]]  ← まだ存在しないノート
→ リンクをクリックすると新規ノートが作成される
→ 未作成ノートはリンクが点線で表示される
```

### 埋め込みリンク
```markdown
![[ノート名]]  ← ノートの内容を埋め込み表示

例:
![[20251113_Obsidianとは何か]]
→ ノート全体が埋め込まれる

![[20251113_Obsidianとは何か#概要]]
→ 特定の見出しのみ埋め込まれる
```

### リンクの検索
- **Ctrl+O**: クイックスイッチャー（ノート検索）
- **Ctrl+Shift+F**: 全文検索（リンクテキストも検索可能）

## メリット・デメリット

### ✅ メリット
1. **知識のネットワーク化**: 孤立した情報を繋げる
2. **発見性**: バックリンクで関連ノートを自動発見
3. **思考の可視化**: グラフビューで全体像を把握
4. **コンテキストの保持**: どこから参照されているかが分かる
5. **ナビゲーション**: 関連ノートへの移動が容易

### ❌ デメリット
1. **リンク過多**: リンクを貼りすぎると読みづらくなる
2. **リンク切れ**: ノート名を変更するとリンクが切れる
3. **循環参照**: 無限ループのリンク構造ができやすい

## よくある問題と解決策

### 問題1: リンクを貼りすぎて読みづらい
**解決策**:
- 1文に1-2個のリンクを目安にする
- 重要な概念のみリンク
- 一般的な単語はリンク不要

**例**:
```markdown
❌ 悪い例:
[[Obsidian]]は[[知識管理]]のための[[ツール]]で、[[Markdown]]ベースの[[ノートアプリ]]です。

✅ 良い例:
[[Obsidian]]は知識管理のためのツールで、Markdownベースのノートアプリです。
```

### 問題2: リンクが切れている
**解決策**:
- Obsidianの「リンク名変更」機能を使用（自動でリンク更新）
- リンク切れチェックプラグインを使用
- 定期的にリンク切れを確認

### 問題3: どのノートにリンクすべきか迷う
**解決策**:
- 明確なルールを設定（例: 概念ノートのみリンク）
- リンク候補をリストアップしてから選択
- バックリンクで既存のリンク構造を確認

## ベストプラクティス

1. **リンクは積極的に**: 知識のネットワーク化を意識
2. **リンク名は明確に**: `[[メモ]]`より`[[20251113_会議メモ]]`の方が明確
3. **エイリアスを活用**: 長いノート名はエイリアスで読みやすく
4. **バックリンクを確認**: 関連ノートを発見する習慣を
5. **グラフビューで確認**: リンク構造の全体像を定期的に確認
