# Obsidianのフロントマター

## 概要
フロントマター（YAML Front Matter）は、Obsidianのノートの先頭に記述するメタデータです。ノートの属性情報（作成日、タグ、カテゴリーなど）を構造化して管理できます。

## 詳細

### フロントマターとは
**フロントマター = ノートの先頭に記述するYAML形式のメタデータ**

```markdown
---
title: ノートのタイトル
created: 2025-11-13
tags: [obsidian, knowledge-management]
status: draft
---

# ノートの本文
ここからノートの内容が始まります。
```

### フロントマターの構造
```
---
キー: 値
キー: 値
キー: [値1, 値2, 値3]
---

ノートの本文
```

## 基本的な使い方

### 基本的なフロントマター
```markdown
---
title: ノートのタイトル
created: 2025-11-13
tags: [tag1, tag2]
---
```

### よく使うフィールド
```markdown
---
title: ノートのタイトル
created: 2025-11-13
updated: 2025-11-14
tags: [tag1, tag2, tag3]
status: draft
priority: high
author: 作成者名
category: カテゴリー
---
```

### フロントマターの編集
1. ノートの先頭に `---` を2行記述
2. その間にYAML形式でメタデータを記述
3. 最後に `---` で閉じる

## 高度な使い方

### 複雑なデータ構造
```markdown
---
title: プロジェクト計画
created: 2025-11-13
tags: [project, planning]
metadata:
  status: active
  priority: high
  deadline: 2025-12-31
  team:
    - メンバーA
    - メンバーB
  milestones:
    - name: フェーズ1
      date: 2025-11-30
    - name: フェーズ2
      date: 2025-12-15
---
```

### Dataviewプラグインとの連携
```markdown
---
created: 2025-11-13
status: active
priority: high
---

# ノートの本文

→ Dataviewプラグインで以下のようにクエリ可能:

```dataview
TABLE status, priority
FROM ""
WHERE status = "active"
SORT priority DESC
```
```

### テンプレートでの活用
```markdown
---
title: "{{title}}"
created: "{{date:YYYY-MM-DD}}"
tags: [{{tags}}]
status: draft
---

# {{title}}

作成日: {{date:YYYY-MM-DD}}
```

## メリット・デメリット

### ✅ メリット
1. **構造化されたメタデータ**: ノートの属性を整理
2. **検索・フィルター**: メタデータで検索・フィルター可能
3. **プラグイン連携**: Dataviewなどと連携
4. **自動化**: テンプレートで自動生成
5. **標準化**: チームで統一したフォーマットを使用可能

### ❌ デメリット
1. **記述の手間**: 毎回フロントマターを書く必要
2. **YAML構文エラー**: 構文ミスでエラーになる
3. **視認性**: ノートの先頭が長くなる

## よくある問題と解決策

### 問題1: YAML構文エラー
**解決策**:
- インデントはスペース2つまたは4つ（統一）
- 文字列はクォートで囲む（必要に応じて）
- リストは `-` または `[]` 形式
- YAMLバリデーターで確認

**例**:
```markdown
❌ 悪い例:
---
title: ノートのタイトル: サブタイトル
tags: tag1, tag2
---

✅ 良い例:
---
title: "ノートのタイトル: サブタイトル"
tags: [tag1, tag2]
---
```

### 問題2: フロントマターが表示される
**解決策**:
- フロントマターは `---` で囲む
- プレビューモードでは自動的に非表示
- エディタモードでは表示される（正常）

### 問題3: どのフィールドを使うべきか迷う
**解決策**:
- テンプレートで標準化
- チームで統一したフォーマットを使用
- 必要最小限のフィールドのみ使用

## ベストプラクティス

1. **標準化**: チームで統一したフォーマットを使用
2. **最小限**: 必要なフィールドのみ記述
3. **テンプレート活用**: よく使うフォーマットはテンプレート化
4. **日付フォーマット**: ISO 8601形式（YYYY-MM-DD）を使用
5. **タグは控えめに**: タグフィールドと本文のタグを統一
