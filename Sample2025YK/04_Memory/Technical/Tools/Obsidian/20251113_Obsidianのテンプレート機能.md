# Obsidianのテンプレート機能

## 概要
Obsidianのテンプレート機能は、よく使うフォーマットを保存して、新しいノート作成時に再利用できる機能です。コアプラグインのTemplatesと、より高度なTemplaterプラグインがあります。

## 詳細

### テンプレートとは
**テンプレート = よく使うフォーマットを保存したファイル**

```
テンプレートの例:
- デイリーノートテンプレート
- 会議メモテンプレート
- プロジェクト概要テンプレート
- 学習ノートテンプレート
```

### テンプレートの種類

#### 1. コアプラグインのTemplates
**Obsidianに標準搭載されている基本的なテンプレート機能**

```
機能:
- テンプレートファイルの保存
- テンプレートの挿入
- 変数の基本的な置換
```

#### 2. Templaterプラグイン
**より高度なテンプレート機能を提供するコミュニティプラグイン**

```
機能:
- 動的な変数の挿入
- 日付の自動生成
- 条件分岐
- ファイル操作
- JavaScriptの実行
```

## 基本的な使い方

### コアプラグインのTemplates

#### テンプレートフォルダの設定
1. 設定を開く（`Ctrl+,` または `Cmd+,`）
2. 「コアプラグイン」→「Templates」を有効化
3. 「テンプレートフォルダの場所」を設定（例: `06_Templates`）

#### テンプレートファイルの作成
1. テンプレートフォルダに新しいファイルを作成
2. テンプレートの内容を記述
3. ファイルを保存

#### テンプレートの挿入
1. ノートを開く
2. `Ctrl+P`（Windows/Linux）または`Cmd+P`（Mac）でコマンドパレットを開く
3. 「Templates: Insert template」を選択
4. テンプレートを選択

### 基本的なテンプレート例

#### デイリーノートテンプレート

## 高度な使い方

### Templaterプラグイン

#### インストール
1. 設定を開く
2. 「コミュニティプラグイン」を選択
3. 「参照」をクリック
4. "Templater"を検索してインストール

#### 基本的な使い方
```markdown
<% tp.file.title %>          ← ファイル名
<% tp.date.now("YYYY-MM-DD") %>  ← 現在の日付
<% tp.file.creation_date("YYYY-MM-DD") %>  ← 作成日
```

#### 高度な機能

##### 条件分岐
```markdown
<% if (tp.file.title.includes("Daily")) { %>
これはデイリーノートです
<% } else { %>
これは通常のノートです
<% } %>
```

##### ファイル操作
```markdown
<% tp.file.move("新しいパス/新しいファイル名.md") %>
```

##### 変数の動的生成
```markdown
<%*
const today = tp.date.now("YYYY-MM-DD");
const week = tp.date.now("YYYY-[W]WW");
-%>
今日は: <% today %>
今週は: <% week %>
```

### テンプレートの変数

#### 日付変数
```markdown
{{date}}              ← 現在の日付（YYYY-MM-DD）
{{date:YYYY-MM-DD}}   ← カスタムフォーマット
{{time}}              ← 現在の時刻
{{date:YYYY-[W]WW}}   ← 週番号
```

#### ファイル変数
```markdown
{{title}}             ← ファイル名（拡張子なし）
{{date:YYYY-MM-DD}}   ← 日付
```

## メリット・デメリット

### ✅ メリット
1. **効率化**: よく使うフォーマットを再利用
2. **標準化**: チームで統一したフォーマットを使用
3. **自動化**: 日付などの自動入力
4. **一貫性**: ノートの形式を統一
5. **時間短縮**: ノート作成時間を短縮

### ❌ デメリット
1. **学習曲線**: Templaterの構文習得に時間がかかる
2. **複雑さ**: 高度なテンプレートは複雑になる
3. **メンテナンス**: テンプレートの更新が必要

## 使用例

### シナリオ1: デイリーノート
```markdown
テンプレート: デイリーノートテンプレート

使用:
1. 新しいデイリーノートを作成
2. テンプレートを挿入
3. 日付が自動入力される
4. フォーマットが統一される
```

## よくある問題と解決策

### 問題1: テンプレートが挿入されない
**解決策**:
- テンプレートフォルダが正しく設定されているか確認
- テンプレートファイルが正しいフォルダにあるか確認
- Templatesプラグインが有効化されているか確認

### 問題2: 変数が置換されない
**解決策**:
- 変数の構文を確認（`{{date}}`など）
- Templaterプラグインを使用している場合、構文を確認（`<% tp.date.now() %>`）
- テンプレートファイルのエンコーディングを確認

### 問題3: テンプレートが複雑すぎる
**解決策**:
- シンプルなテンプレートから始める
- 必要に応じて機能を追加
- テンプレートを分割（複数のテンプレートに分ける）

## ベストプラクティス

1. **シンプルに**: 必要最小限の要素のみ含める
2. **標準化**: チームで統一したテンプレートを使用
3. **段階的に**: シンプルなテンプレートから始める
4. **定期的に見直す**: 使われていないテンプレートを削除
5. **ドキュメント化**: テンプレートの使い方を記録
