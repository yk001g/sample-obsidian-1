# Obsidianのインストールとセットアップ

## 概要
Obsidianのインストールから基本的なセットアップまで、初めて使う人向けの完全ガイドです。Vaultの作成、基本設定、推奨プラグインのインストールまでをカバーします。

## 詳細

### Obsidianとは
Obsidianは、ローカルで動作するMarkdownベースの知識管理アプリケーションです。詳細は[[20251113_Obsidianとは何か]]を参照してください。

## 基本的な使い方

### ステップ1: インストール

#### Windows/Mac/Linux
1. [Obsidian公式サイト](https://obsidian.md)にアクセス
2. 「Download」をクリック
3. オペレーティングシステムを選択
4. インストーラーをダウンロード
5. インストーラーを実行してインストール

#### モバイル（iOS/Android）
1. App Store（iOS）またはGoogle Play（Android）で「Obsidian」を検索
2. アプリをインストール
3. 初回起動時にアカウント作成（オプション）

### ステップ2: Vaultの作成

#### デスクトップ版
1. Obsidianを起動
2. 「Create new vault」を選択
3. Vault名を入力（例: `SecondBrain`）
4. 保存場所を選択（例: `~/Documents/SecondBrain`）
5. 「Create」をクリック

#### モバイル版
1. アプリを起動
2. 「Create new vault」を選択
3. Vault名を入力
4. 保存場所を選択（ローカルまたはクラウド）

### ステップ3: 基本設定

#### 設定の開き方
- `Ctrl+,`（Windows/Linux）または`Cmd+,`（Mac）
- または、メニューから「Settings」を選択

#### 推奨設定

##### 1. エディタ設定
```
- 行番号を表示: ON
- 折り返し: ON
- フォントサイズ: 14-16px
- フォント: 等幅フォント（例: Fira Code, JetBrains Mono）
```

##### 2. ファイルとリンク設定
```
- 新しいリンクの形式: 最短パス
- デフォルトの場所: 現在のフォルダ
- 自動更新内部リンク: ON
```

##### 3. ホットキー設定
```
よく使うホットキー:
- Ctrl+O: クイックスイッチャー
- Ctrl+Shift+F: 全文検索
- Ctrl+G: グラフビュー
- Ctrl+P: コマンドパレット
```

### ステップ4: コアプラグインの有効化

#### 推奨コアプラグイン
1. **Daily Notes**: デイリーノート機能
2. **Templates**: テンプレート機能
3. **Graph View**: グラフビュー
4. **Backlinks**: バックリンクパネル
5. **Tag Pane**: タグパネル
6. **Search**: 検索機能
7. **File Recovery**: ファイル回復機能

#### 有効化手順
1. 設定を開く
2. 「コアプラグイン」を選択
3. 必要なプラグインを有効化

### ステップ5: コミュニティプラグインのインストール

#### 安全モードの無効化
1. 設定を開く
2. 「コミュニティプラグイン」を選択
3. 「安全モードをオフにする」を有効化

#### 推奨プラグイン

##### 必須プラグイン
1. **Dataview**: データベース機能
2. **Templater**: 高度なテンプレート機能
3. **Calendar**: カレンダービュー

##### 便利なプラグイン
1. **Tasks**: タスク管理
2. **Kanban**: カンバンボード
3. **Excalidraw**: 図形描画
4. **Omnisearch**: 高度な検索

#### インストール手順
1. 「参照」をクリック
2. プラグイン名を検索
3. 「インストール」をクリック
4. インストール後、有効化

## 高度な使い方

### ワークスペースの設定
```
ワークスペース = パネルの配置とサイズを保存

使い方:
1. パネルを配置
2. メニュー → Workspaces → Save workspace
3. 名前を付けて保存
4. 後で「Load workspace」で読み込み
```

### テーマの変更
1. 設定を開く
2. 「外観」を選択
3. 「テーマ」から選択
4. または、「コミュニティテーマ」からインストール

### カスタムCSS
1. 設定を開く
2. 「外観」を選択
3. 「CSSスニペット」を選択
4. 新しいスニペットを作成
5. CSSを記述

### バックアップの設定

#### Gitでバックアップ
```bash
# VaultフォルダでGitを初期化
cd ~/Documents/SecondBrain
git init
git add .
git commit -m "Initial commit"

# 定期的なバックアップ
git add .
git commit -m "Daily backup"
```

#### クラウドストレージでバックアップ
- Dropbox、iCloud、Google DriveなどのフォルダにVaultを配置
- 自動的に同期される

## メリット・デメリット

### ✅ メリット
1. **完全無料**: 個人利用は無料
2. **ローカル保存**: データが自分のPCに保存
3. **プライバシー**: データが外部に送信されない
4. **拡張性**: プラグインで無限に拡張可能
5. **カスタマイズ**: テーマ、CSS、プラグインで自由にカスタマイズ

### ❌ デメリット
1. **学習曲線**: 初期設定に時間がかかる
2. **モバイル同期**: 公式の同期サービスは有料
3. **設定の複雑さ**: 多くの設定項目がある

## 使用例

### シナリオ1: 個人の知識管理
```
セットアップ:
1. Vault作成: Personal-Knowledge-Base
2. フォルダ構造: 00_Memo, 01_Inbox, 02_Daily, 04_Memory
3. プラグイン: Dataview, Templater, Calendar
4. テンプレート: デイリーノート、学習ノート

→ 個人の知識管理システムを構築
```

### シナリオ2: チームでの使用
```
セットアップ:
1. Vault作成: Team-Knowledge-Base
2. 共有設定: クラウドストレージで共有
3. テンプレート: 統一したフォーマット
4. プラグイン: チームで統一したプラグイン

→ チームの知識管理システムを構築
```

## よくある問題と解決策

### 問題1: Vaultが開けない
**解決策**:
- Vaultのパスを確認
- ファイルのアクセス権限を確認
- Obsidianを再起動

### 問題2: プラグインが動作しない
**解決策**:
- プラグインが有効化されているか確認
- プラグインのバージョンを確認
- Obsidianを再起動

### 問題3: 設定が反映されない
**解決策**:
- Obsidianを再起動
- 設定を再度確認
- キャッシュをクリア

## ベストプラクティス

1. **段階的に設定**: 一度にすべてを設定しない
2. **バックアップ**: 定期的にバックアップを取る
3. **プラグインは控えめに**: 本当に必要なプラグインだけインストール
4. **設定の記録**: 重要な設定をメモしておく
5. **コミュニティ活用**: フォーラムやDiscordで情報収集
