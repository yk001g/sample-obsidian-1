# Obsidianの検索機能

## 概要
Obsidianの検索機能は、ノート内のテキスト、タグ、リンク、フロントマターなど、あらゆる情報を検索できる強力なツールです。高度な検索演算子を使うことで、精密な検索が可能です。

## 詳細

### 検索の種類
Obsidianには以下の検索方法があります：

1. **クイックスイッチャー**: ノート名の検索
2. **全文検索**: ノート内容の検索
3. **高度な検索**: 検索演算子を使った精密な検索

### 検索の開き方
- **クイックスイッチャー**: `Ctrl+O`（Windows/Linux）または`Cmd+O`（Mac）
- **全文検索**: `Ctrl+Shift+F`（Windows/Linux）または`Cmd+Shift+F`（Mac）
- **コマンドパレット**: `Ctrl+P`（Windows/Linux）または`Cmd+P`（Mac）

## 基本的な使い方

### クイックスイッチャー
```
Ctrl+O または Cmd+O

機能:
- ノート名の検索
- 最近開いたノートの表示
- ノートへの直接移動

使い方:
1. Ctrl+Oを押す
2. ノート名の一部を入力
3. 候補から選択してEnter
```

### 全文検索
```
Ctrl+Shift+F または Cmd+Shift+F

機能:
- すべてのノート内のテキストを検索
- 検索結果のハイライト表示
- 検索結果の一覧表示

使い方:
1. Ctrl+Shift+Fを押す
2. 検索語を入力
3. 検索結果を確認
```

## 高度な使い方

### 検索演算子

#### 基本的な演算子
```markdown
# タグ検索
tag:#タグ名

# フォルダ検索
path:フォルダ名

# ファイル名検索
file:ファイル名

# リンク検索
link:[[ノート名]]

# バックリンク検索
-backlink:[[ノート名]]
```

#### 複合検索
```markdown
# 複数の条件を組み合わせ
tag:#react path:04_Memory

# 除外検索
tag:#react -tag:#hooks

# 正規表現
/正規表現/

# 大文字小文字を区別
case:検索語
```

### 検索の例

#### 例1: 特定のタグを持つノート
```
tag:#react
→ #reactタグを持つすべてのノート
```

#### 例2: 特定のフォルダ内のノート
```
path:04_Memory/AI
→ 04_Memory/AIフォルダ内のすべてのノート
```

#### 例3: リンクを含むノート
```
link:[[Obsidian]]
→ [[Obsidian]]へのリンクを含むノート
```

#### 例4: 複合条件
```
tag:#react path:04_Memory -tag:#hooks
→ #reactタグを持ち、04_Memoryフォルダ内にあり、#hooksタグを持たないノート
```

### 検索結果の操作
```
検索結果パネルには以下が表示される:

1. マッチしたノートの一覧
2. マッチした行のプレビュー
3. マッチ数

操作:
- ノートをクリック: 該当ノートに移動
- 行をクリック: 該当行に移動
- ハイライト: 検索語がハイライト表示
```

## メリット・デメリット

### ✅ メリット
1. **高速検索**: 大量のノートでも高速に検索
2. **柔軟な検索**: 検索演算子で精密な検索が可能
3. **ハイライト表示**: 検索語がハイライト表示
4. **複合検索**: 複数の条件を組み合わせ可能
5. **正規表現対応**: 高度なパターンマッチング

### ❌ デメリット
1. **学習曲線**: 検索演算子の習得に時間がかかる
2. **検索結果の多さ**: 検索結果が多すぎると見づらい
3. **パフォーマンス**: 大量のノートがあると検索が遅い

## 使用例

### シナリオ1: 学習ノートの検索
```
検索: tag:#react path:04_Memory
→ React関連の学習ノートを一括で検索
```

### シナリオ2: プロジェクト管理
```
検索: tag:#project/project-a status:active
→ プロジェクトAのアクティブなノートを検索
```

### シナリオ3: リンクの確認
```
検索: link:[[Obsidian]]
→ Obsidianへのリンクを含むノートを検索
```

## よくある問題と解決策

### 問題1: 検索結果が多すぎる
**解決策**:
- 検索条件を絞り込む（タグ、フォルダを追加）
- 除外検索を使う（-演算子）
- 検索範囲を限定する

### 問題2: 検索演算子が分からない
**解決策**:
- 基本的な演算子から覚える
- 検索パネルのヘルプを確認
- 段階的に高度な検索を学ぶ

### 問題3: 検索が遅い
**解決策**:
- 検索範囲を限定（フォルダ指定）
- インデックスの再構築
- 不要なノートをアーカイブ

## ベストプラクティス

1. **検索条件を明確に**: 何を探しているか明確にする
2. **タグとフォルダを活用**: 検索を効率化
3. **検索履歴を活用**: よく使う検索を保存
4. **正規表現を学ぶ**: 高度な検索が可能
5. **検索結果を整理**: 検索結果から新しいノートを作成
