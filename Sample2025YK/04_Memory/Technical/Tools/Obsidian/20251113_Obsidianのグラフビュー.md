# Obsidianのグラフビュー

## 概要
グラフビューは、Obsidianのノート間のリンク関係を視覚的に表示する機能です。知識のネットワーク全体を俯瞰し、ノート間の関係性を理解するのに役立ちます。

## 詳細

### グラフビューとは
**グラフビュー = ノート間のリンク関係を視覚化したネットワーク図**

```
各ノート = ノード（円）
リンク = エッジ（線）

ノードの大きさ = リンク数の多さ
ノードの色 = カテゴリーやタグ
ノード間の距離 = リンクの強さ
```

### グラフビューの開き方
1. 左サイドバーの「グラフビュー」アイコンをクリック
2. または、`Ctrl+G`（Windows/Linux）または`Cmd+G`（Mac）

## 基本的な使い方

### グラフの操作
- **パン**: マウスドラッグで移動
- **ズーム**: マウスホイールで拡大・縮小
- **ノードをクリック**: ノートを開く
- **ノードをドラッグ**: ノードの位置を調整

### フィルター機能
```
グラフビューには以下のフィルターがある:

1. タグフィルター
   - 特定のタグを持つノートのみ表示

2. フォルダフィルター
   - 特定のフォルダ内のノートのみ表示

3. リンクフィルター
   - 特定のノートにリンクしているノートのみ表示

4. 検索フィルター
   - 検索クエリに一致するノートのみ表示
```

### ノードの表示設定
- **ノードサイズ**: リンク数に応じて自動調整
- **ノード色**: タグやフォルダに応じて色分け
- **ラベル表示**: ノート名の表示/非表示

## 高度な使い方

### ローカルグラフ
```
ローカルグラフ = 特定のノートを中心とした部分グラフ

使い方:
1. ノートを開く
2. 右サイドバーの「ローカルグラフ」を開く
3. そのノートと直接リンクしているノートのみ表示

設定:
- 深さ: 何階層まで表示するか（1-3階層が一般的）
- ノード数: 表示するノート数の上限
```

### グラフのカスタマイズ
```
グラフビューの設定:

1. ノードサイズ
   - 固定サイズ
   - リンク数に応じたサイズ

2. ノード色
   - タグに応じた色
   - フォルダに応じた色
   - カスタム色

3. エッジ（リンク線）
   - 太さ
   - 色
   - アニメーション

4. レイアウト
   - フォース指向レイアウト（デフォルト）
   - 階層レイアウト
```

### グラフの分析
```
グラフビューで分かること:

1. ハブノート（多くのリンクを持つノート）
   - 知識の中心となるノート
   - 重要な概念ノート

2. 孤立ノート（リンクがないノート）
   - まだ繋がっていないノート
   - リンクを追加する候補

3. クラスター（ノートのグループ）
   - 関連するノートのグループ
   - トピックの境界

4. ブリッジ（異なるクラスターを繋ぐノート）
   - 異なるトピックを繋ぐ重要なノート
```

## メリット・デメリット

### ✅ メリット
1. **全体像の把握**: 知識のネットワーク全体を俯瞰
2. **関係性の可視化**: ノート間の関係性を視覚的に理解
3. **発見性**: 思わぬ関連性を発見できる
4. **ナビゲーション**: グラフから直接ノートに移動
5. **分析**: ハブノートや孤立ノートを発見

### ❌ デメリット
1. **パフォーマンス**: 大量のノートがあると動作が重い
2. **視認性**: ノート数が多いと見づらい
3. **学習曲線**: グラフの読み方が分からないと混乱

## 使用例

### シナリオ1: 学習ノートの構造分析
```
グラフビューで確認すると:
- 「React基礎」ノートがハブになっている
- 「Hooks」「コンポーネント」「状態管理」がクラスターを形成
- 「実践プロジェクト」がブリッジになっている

→ 学習の全体像と進捗を把握できる
```

### シナリオ2: プロジェクト管理
```
グラフビューで確認すると:
- 「プロジェクトA」ノートが中心
- 「タスク1」「タスク2」「タスク3」が周囲に配置
- 「クライアントB」が別のクラスターを形成

→ プロジェクトの構造と関連情報を把握できる
```

### シナリオ3: 知識ベースの成長
```
1ヶ月前のグラフ:
- 10個のノート、5個のリンク

1ヶ月後のグラフ:
- 50個のノート、100個のリンク
- ハブノートが明確に
- クラスターが形成されている

→ 知識ベースの成長を視覚的に確認できる
```

## よくある問題と解決策

### 問題1: グラフが重くて動作が遅い
**解決策**:
- フィルターで表示ノート数を減らす
- ローカルグラフを使用
- ノード数を制限する設定

### 問題2: ノートが多すぎて見づらい
**解決策**:
- タグやフォルダでフィルター
- ズームアウトして全体を見る
- 重要なノートのみ表示

### 問題3: グラフの意味が分からない
**解決策**:
- 小さなグラフから始める
- ハブノートと孤立ノートを意識
- クラスターの形成を観察

## ベストプラクティス

1. **定期的にグラフを確認**: 知識の構造を把握する習慣を
2. **ハブノートを意識**: 重要な概念ノートを中心に
3. **孤立ノートを減らす**: リンクを追加してネットワーク化
4. **フィルターを活用**: 特定のトピックに焦点を当てる
5. **グラフの成長を記録**: 定期的にスクリーンショットを撮る
