# Obsidianのタグ機能

## 概要
タグは、Obsidianでノートを横断的に分類する機能です。フォルダとは異なり、1つのノートに複数のタグを付けることができ、柔軟な分類が可能です。

## 詳細

### タグとは
**タグ = ノートに付ける横断的な分類ラベル**

```
フォルダ = 階層的分類（1つのノートは1つのフォルダに）
タグ = 横断的分類（1つのノートに複数のタグを付けられる）

例:
ノート: "React Hooks学習メモ"
フォルダ: 04_Memory/Technical/Programming/
タグ: #react #hooks #learning #frontend
```

### タグの記述方法
```markdown
# 基本的なタグ
#タグ名

# 階層タグ
#カテゴリー/サブカテゴリー
例: #ai/tools #ai/techniques

# フロントマターでのタグ
---
tags: [tag1, tag2, tag3]
---
```

## 基本的な使い方

### タグの作成
1. ノート内で `#` を入力
2. タグ名を入力（自動補完が表示される）
3. スペースまたは改行で確定

### タグの検索
- **タグパネル**: 左サイドバーの「タグ」パネル
- **検索**: `tag:#タグ名` で検索
- **クイックスイッチャー**: `Ctrl+O` → `#タグ名`

### タグの編集
- **タグ名の変更**: タグをクリック → 名前を変更（全ノートに反映）
- **タグの削除**: ノートから `#タグ名` を削除

## 高度な使い方

### 階層タグ
```markdown
# 階層タグの例
#ai/tools/cursor
#ai/tools/claude
#ai/techniques/prompting
#ai/techniques/rag

→ タグパネルで階層的に表示される
```

### タグの自動補完
```
# タグを入力すると:
#react と入力
→ #react
→ #react-hooks
→ #react-native
などが候補として表示される
```

### タグの統計
```
タグパネルには以下が表示される:

1. タグ名
2. 使用回数（ノート数）
3. 階層構造

→ よく使うタグが一目で分かる
```

### タグとフォルダの使い分け
```
フォルダを使う場面:
- プロジェクト単位の分類
- 期間単位の分類（2025/2025-11など）
- 明確な階層構造が必要な場合

タグを使う場面:
- 横断的な分類（#重要 #ToDo #アイデア）
- 複数のカテゴリーに属する場合
- 柔軟な分類が必要な場合
```

## メリット・デメリット

### ✅ メリット
1. **柔軟な分類**: 1つのノートに複数のタグを付けられる
2. **横断的検索**: タグでノートを横断的に検索
3. **階層構造**: 階層タグで細かく分類可能
4. **自動補完**: 既存のタグを自動補完
5. **統計**: タグの使用頻度を確認

### ❌ デメリット
1. **タグ過多**: タグを付けすぎると管理が大変
2. **一貫性**: タグ名の統一が難しい（#react vs #React）
3. **検索性**: タグが多すぎると検索が困難
4. **視認性**: タグが多すぎるとノートが見づらい

## 使用例

### シナリオ1: 学習ノートの分類
```markdown
# React Hooks学習メモ

#react #hooks #learning #frontend #javascript

→ 複数の観点で分類できる
```

### シナリオ2: プロジェクト管理
```markdown
# プロジェクトAのタスク

#project/project-a #status/todo #priority/high

→ プロジェクト、ステータス、優先度で分類
```

### シナリオ3: アイデア管理
```markdown
# 新機能のアイデア

#idea #feature #enhancement #status/draft

→ アイデアの種類とステータスで分類
```

## よくある問題と解決策

### 問題1: タグを付けすぎて管理できない
**解決策**:
- タグは控えめに（3-5個程度）
- 重要な分類のみタグを使用
- 定期的にタグを見直す

**例**:
```markdown
❌ 悪い例:
#react #hooks #useState #useEffect #learning #frontend #javascript #web #development #programming

✅ 良い例:
#react #hooks #learning
```

### 問題2: タグ名の統一ができない
**解決策**:
- タグ命名規則を設定（例: 小文字、ハイフン区切り）
- 既存のタグを確認してから作成
- タグの統合・リネーム機能を活用

**例**:
```markdown
統一ルール:
- 小文字のみ使用
- 複数単語はハイフン区切り
- 略語は避ける

例: #react-hooks（✅） vs #ReactHooks（❌）
```

### 問題3: タグとフォルダの使い分けが分からない
**解決策**:
- フォルダ: 明確な階層構造が必要な場合
- タグ: 横断的な分類が必要な場合
- ハイブリッド: フォルダ + タグの組み合わせ

## ベストプラクティス

1. **タグは控えめに**: 3-5個程度を目安
2. **命名規則を統一**: チームで統一したルールを設定
3. **階層タグを活用**: 細かく分類したい場合
4. **定期的に見直す**: 使われていないタグを削除
5. **フォルダと組み合わせ**: フォルダ + タグのハイブリッド戦略
