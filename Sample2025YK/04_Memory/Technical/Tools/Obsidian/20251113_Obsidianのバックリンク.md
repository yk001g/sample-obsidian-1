# Obsidianのバックリンク

## 概要
バックリンクは、あるノートを参照している他のノートを自動的に表示する機能です。双方向リンクの「逆方向」のリンクを可視化し、知識の文脈を理解するのに役立ちます。

## 詳細

### バックリンクとは
**バックリンク = このノートを参照しているノートの一覧**

```
ノートA → ノートB（リンク）
ノートC → ノートB（リンク）
ノートD → ノートB（リンク）

→ ノートBのバックリンクには、ノートA、C、Dが表示される
```

### バックリンクの表示場所
1. **右サイドバー**: 「バックリンク」パネル
2. **ノート内**: `[[ノート名]]`のリンクをクリックすると表示
3. **グラフビュー**: 視覚的にバックリンクを確認

## 基本的な使い方

### バックリンクパネルの開き方
1. ノートを開く
2. 右サイドバーの「バックリンク」アイコンをクリック
3. または、`Ctrl+Shift+B`（Windows/Linux）または`Cmd+Shift+B`（Mac）

### バックリンクの確認
```
バックリンクパネルには以下が表示される:

1. リンク元ノートの一覧
   - ノート名
   - リンクが含まれる行のプレビュー
   - クリックで該当ノートに移動

2. 未リンク参照
   - ノート名は書かれているが、リンクになっていない
   - 例: "Obsidianについて"（[[Obsidian]]ではない）
```

### バックリンクからの移動
- **ノート名をクリック**: リンク元ノートに移動
- **プレビューをクリック**: リンクが含まれる行に移動

## 高度な使い方

### 未リンク参照の活用
```
未リンク参照 = ノート名は書かれているが、リンクになっていない参照

例:
"Obsidianについて調べた"
→ 「Obsidian」という文字列はあるが、[[Obsidian]]ではない

→ 未リンク参照をクリックすると、自動的にリンクに変換できる
```

### バックリンクの検索
- **バックリンクパネル内で検索**: 特定のノートを探す
- **フィルター**: リンク元ノートを絞り込み

### バックリンクのコンテキスト表示
```
バックリンクパネルには以下が表示される:

1. リンク元ノート名
2. リンクが含まれる行の前後（コンテキスト）
3. リンクの種類（通常リンク、埋め込みリンクなど）
```

## メリット・デメリット

### ✅ メリット
1. **文脈の理解**: このノートがどこで使われているかが分かる
2. **関連ノートの発見**: 思わぬ関連性を発見できる
3. **知識のネットワーク化**: ノート間の関係性を把握
4. **ナビゲーション**: 関連ノートへの移動が容易
5. **未リンク参照の発見**: リンクにすべき参照を見つけられる

### ❌ デメリット
1. **情報過多**: バックリンクが多すぎると見づらい
2. **パフォーマンス**: 大量のバックリンクがあると動作が重い
3. **未リンク参照の誤検出**: 意図しない文字列が検出される

## 使用例

### シナリオ1: 学習ノートの復習
```

- 「useStateの使い方」ノートから参照されている
- 「カスタムフック作成」ノートから参照されている
- 「実践プロジェクト」ノートから参照されている

→ 関連する学習ノートを一括で確認できる
```

```
→ クライアントAに関連するすべての情報を把握できる
```

```
- 「開発タスク」ノートから参照されている

```

## よくある問題と解決策

### 問題1: バックリンクが多すぎて見づらい
**解決策**:
- バックリンクパネルを折りたたむ
- 検索機能で絞り込み
- 重要なリンクのみ表示する設定

### 問題2: 未リンク参照が多すぎる
**解決策**:
- 重要な参照のみリンクに変換
- 一般的な単語は無視
- 定期的に未リンク参照を確認

### 問題3: バックリンクが表示されない
**解決策**:
- バックリンクパネルが開いているか確認
- ノートが正しくリンクされているか確認
- Obsidianを再起動
