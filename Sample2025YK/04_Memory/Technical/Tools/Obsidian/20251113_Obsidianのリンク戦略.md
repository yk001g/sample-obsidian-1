# Obsidianのリンク戦略

## 概要
Obsidianのリンク戦略は、知識をネットワーク化するための重要な要素です。適切なリンク戦略を実践することで、知識の関係性が明確になり、新しい洞察や発見が生まれます。

## 詳細

### リンク戦略の基本原則

#### 原則1: リンクは積極的に
```
✅ 良い例:
[[React]]は、ユーザーインターフェースを構築するための
JavaScriptライブラリです。[[コンポーネント]]ベースの
アーキテクチャを採用しています。

❌ 悪い例:
Reactは、ユーザーインターフェースを構築するための
JavaScriptライブラリです。コンポーネントベースの
アーキテクチャを採用しています。
（リンクがない）
```

#### 原則2: リンクの粒度を意識
```
✅ 良い例:
1文に1-2個のリンク

[[React]]は、ユーザーインターフェースを構築するための
JavaScriptライブラリです。

❌ 悪い例:
リンク過多

[[React]]は、[[ユーザーインターフェース]]を[[構築]]するための
[[JavaScript]][[ライブラリ]]です。
```

#### 原則3: 重要な概念のみリンク
```
✅ リンクすべきもの:
- 重要な概念
- 関連するノート
- 前提知識
- 具体例

❌ リンク不要なもの:
- 一般的な単語
- 1回しか出てこない単語
- 確定していない概念
```

## 基本的な使い方

### リンクの種類

#### 1. 基本的なリンク
```markdown
[[ノート名]]
```

#### 2. エイリアス（表示名の変更）
```markdown
[[ノート名|表示名]]
例: [[20251113_Obsidianとは何か|Obsidianの基本]]
```

#### 3. 見出しへのリンク
```markdown
[[ノート名#見出し名]]
例: [[20251113_Obsidianとは何か#概要]]
```

#### 4. ブロックへのリンク
```markdown
[[ノート名#^block-id]]
例: [[20251113_Obsidianとは何か#^abc123]]
```

### リンクの作成タイミング

#### 書きながらリンク（推奨）
```markdown
[[React]]を使うと、[[コンポーネント]]ベースの
アプリケーションを構築できます。
```

#### 後からリンク（OK）
```markdown
Reactを使うと、コンポーネントベースの
アプリケーションを構築できます。

↓（後で修正）

[[React]]を使うと、[[コンポーネント]]ベースの
アプリケーションを構築できます。
```

## 高度な使い方

### リンク構造の設計

#### ハブノート戦略
```
ハブノート = 多くのリンクを持つ中心的なノート

例:
「React基礎」ノート
→ 「useState」「useEffect」「useContext」などにリンク
→ ハブノートとして機能
```

#### クラスター戦略
```
クラスター = 関連するノートのグループ

例:
「React Hooks」クラスター
- React Hooks基礎
- useStateの使い方
- useEffectの使い方
- useContextの使い方
→ 互いにリンクで繋がっている
```

#### ブリッジノート戦略
```
ブリッジノート = 異なるクラスターを繋ぐノート

例:
「フロントエンド開発」ノート
→ 「React」クラスターと「Vue」クラスターを繋ぐ
```

### リンクの品質管理

#### リンク切れのチェック
```
定期的にリンク切れをチェック:
1. ノート名を変更した場合
2. ノートを移動した場合
3. ノートを削除した場合

→ Obsidianの「リンク名変更」機能を使用
→ 自動でリンクが更新される
```

#### 孤立ノートの発見
```
グラフビューで孤立ノートを確認:
1. グラフビューを開く
2. リンクがないノートを確認
3. 適切なリンクを追加

→ 知識のネットワーク化を促進
```

## メリット・デメリット

### ✅ メリット
1. **知識のネットワーク化**: 孤立した情報を繋げる
2. **発見性**: バックリンクで関連ノートを自動発見
3. **思考の可視化**: グラフビューで全体像を把握
4. **コンテキストの保持**: どこから参照されているかが分かる
5. **ナビゲーション**: 関連ノートへの移動が容易

### ❌ デメリット
1. **リンク過多**: リンクを貼りすぎると読みづらくなる
2. **リンク切れ**: ノート名を変更するとリンクが切れる
3. **循環参照**: 無限ループのリンク構造ができやすい

## 使用例

### シナリオ1: 学習ノートのネットワーク化
```markdown
# React Hooks基礎

[[useState]]を使うと状態管理ができる。
[[useEffect]]でライフサイクルを制御できる。
[[useContext]]でコンテキストを共有できる。

→ 各フックのノートが自動的にバックリンクで繋がる
```

### シナリオ2: プロジェクト管理
```markdown
# プロジェクトA

関連する[[クライアントB]]との打ち合わせ。
[[タスク1]]と[[タスク2]]を完了。

→ プロジェクト、クライアント、タスクがすべて繋がる
```

### シナリオ3: アイデアの展開
```markdown
# 新機能のアイデア

[[既存機能A]]を拡張して...
[[競合ツールB]]の機能を参考に...
[[ユーザーフィードバックC]]を反映して...

→ アイデアの背景や関連情報がすべて繋がる
```

## よくある問題と解決策

### 問題1: リンクを貼りすぎて読みづらい
**解決策**:
- 1文に1-2個のリンクを目安にする
- 重要な概念のみリンク
- 一般的な単語はリンク不要

### 問題2: リンクが切れている
**解決策**:
- Obsidianの「リンク名変更」機能を使用（自動でリンク更新）
- リンク切れチェックプラグインを使用
- 定期的にリンク切れを確認

### 問題3: どのノートにリンクすべきか迷う
**解決策**:
- 明確なルールを設定（例: 概念ノートのみリンク）
- リンク候補をリストアップしてから選択
- バックリンクで既存のリンク構造を確認

## ベストプラクティス

1. **リンクは積極的に**: 知識のネットワーク化を意識
2. **リンク名は明確に**: `[[メモ]]`より`[[20251113_会議メモ]]`の方が明確
3. **エイリアスを活用**: 長いノート名はエイリアスで読みやすく
4. **バックリンクを確認**: 関連ノートを発見する習慣を
5. **グラフビューで確認**: リンク構造の全体像を定期的に確認
6. **リンクの品質管理**: リンク切れや孤立ノートを定期的にチェック
