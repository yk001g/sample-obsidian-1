## 🚀 コマンドの実行方法

### 実行方法

Cursorでは以下の3つの方法でコマンドを実行できます：

#### 方法1: Cursor Chat (Cmd/Ctrl + L)
チャットを開いて、`/コマンド名` を入力します。

```
/cmd + L → /daily
```

#### 方法2: インライン (Cmd/Ctrl + K)
インライン編集で `/コマンド名` を入力します。

```
/cmd + K → /daily
```

#### 方法3: コンポーザー (Cmd/Ctrl + I)
コンポーザーで `/コマンド名` を入力します。

```
/cmd + I → /daily
```

### コマンド補完

- コマンド名の最初の数文字を入力すると補完されます
- ファイル名の引数がない場合、現在開いているファイルが対象になります
- 各コマンドは対話形式で実行され、確認や選択を求められる場合があります

## 📅 日次コマンド

### `/daily` - デイリーノート作成

**説明**: 今日のデイリーノートを作成します（朝のルーティン5分）。

**実行内容**:
- ファイル名: `02_Daily/YYYY/YYYY-MM/YYYY-MM-DD/YYYY-MM-DD-Daily.md`（今日の日付を使用）
- 今日の日付を自動挿入
- Inboxフォルダ（`01_Inbox/`）から未完了タスクを3つ抽出
- 前日のDailyへのリンクを追加
- テンプレートを使用: `06_Templates/Daily/デイリーノートテンプレート.md`

**使用例**:
```
/daily
```

**出力例**:
```markdown
---
date: 2025-11-05
day: Tuesday
week: 2025-W45
tags: [daily, 2025, 2025-11]
---

# 2025-11-05 - Tuesday

## 🔋 Today's Energy & Focus
- Energy Level: 8/10 ⚡
- Focus Quality: High 🎯
- Mood: 😊

## ⭐ Top 3 Priorities
- [ ] [Inboxから抽出したタスク1]
- [ ] [Inboxから抽出したタスク2]
- [ ] [Inboxから抽出したタスク3]
```

**ヒント**:
- 「Top 3 Priorities」は簡潔に
- 「Evening Reflection」は夕方に追記する想定
- 関連プロンプト: [[08_prompts/02_生成系/20250113_Daily作成.md]]

2. 3ヶ月以上前のファイルを抽出

## 🗂️ 整理・管理コマンド

### `/memo-to-inbox [ファイル名]` - Memo→Inbox変換

**説明**: 00_Memoの雑なメモを全て01_Inboxの形式に整形して変換します（承認不要で自動実行）。

**重要**: 全てのファイルの中身を必ず読み込んで、内容を丁寧に分析してから整形します。

**引数**:
- Memoファイル名（省略可）: 変換元のファイル。省略時は00_Memo内の全ファイルをリスト表示

**実行内容**:
1. 00_Memoのメモを全て読み込む
2. 内容を分析して整理
3. 日付フォーマット `YYYYMMDD_内容.md` でファイル名を生成
4. 内容を整形して01_Inbox形式に変換
5. **自動的に** `01_Inbox/` に作成
6. 元のファイルは `99_Archive/` に移動
7. 変換後に00_Memo内の空のフォルダを自動削除

**使用例**:
```
/memo-to-inbox                    # 00_Memo内の全ファイルをリスト表示
/memo-to-inbox 雑メモ.md         # 特定のファイルを変換
/memo-to-inbox 学んだこと.txt     # 拡張子が違ってもOK
```

**変換ルール**:
- ファイル名: 内容からタイトルを生成（形式: `YYYYMMDD_内容.md`）
- 内容構成: 概要、内容、次アクションを含む
- トピック分割: 複数のトピックが含まれている場合、分割する

**関連プロンプト**: [[08_prompts/01_整理系/20250113_Memo→Inbox変換.md]]

### `/organize-inbox [ファイル名]` - Inbox整理・振り分け支援

**説明**: Inboxフォルダ内のファイルを適切なフォルダに振り分けます。

**重要**: 全てのファイルの中身を必ず読み込んで、内容を丁寧に分析・理解・抽象化してから振り分けます。

**引数**:
- ファイル名: 振り分け対象のファイル（ユーザーが指定）

**実行内容**:
1. **必ずファイルの中身を全内容読み込む**
2. **内容を理解して抽象化する**
3. **内容から振り分け先を判定**
4. **適切なフォルダへの振り分けを実行**
5. **移動後、必要に応じてメタデータを更新**

**振り分け先**:
- `04_Memory/`: 3ヶ月後も参照する価値がある知識（品質スコア7点以上）
- `03_Input/`: 今週〜今月使う資料
- `05_Output/Projects/`: 期限ありのプロジェクト
- `05_Output/Areas/`: 継続的な活動エリア
- `02_Daily/`: 日次記録
- `99_Archive/`: 判断に迷う場合

**使用例**:
```
/organize-inbox                    # 全ファイルを振り分け
/organize-inbox 20241022_API設計メモ.md
```

**振り分け判定基準**:
- Memory候補: 自分の言葉で書かれている、1ノート1アイデア、完結性、適切なリンク・タグ、3ヶ月後も価値あり
- Input候補: 今週〜今月使う資料
- Output候補: プロジェクト・継続活動関連
- Daily候補: 日次記録

**関連プロンプト**: [[08_prompts/01_整理系/20250113_Inbox整理.md]]

### `/inbox-review` - Inbox整理

**説明**: Inboxを分析して整理案を提示します。

**実行内容**:
1. `01_Inbox/`内の全ファイルをスキャン
2. 各ノートを以下の基準で評価（10点満点）:
   - Memory候補 (7点以上、3ヶ月後も価値あり)
   - Input候補 (今週〜今月使う)
   - Output候補 (プロジェクト・継続活動)
   - Daily候補 (日次記録)
   - 要追加情報 (4-6点)
   - 削除候補 (3点以下)

**評価基準**:
- 自分の言葉で書かれているか: 3点
- 1ノート1アイデアか: 2点
- 完結性: 2点
- リンクがあるか: 2点
- タグがあるか: 1点

```
/inbox-review
```

### Input候補（03_Input/へ）
1. [ノート名] (スコア: 6/10)
   理由: 今週使う資料

### Output候補（05_Output/へ）
1. [ノート名] (スコア: 7/10)
   理由: プロジェクト関連

### 要追加情報
1. [ノート名] (スコア: 5/10)
   不足: 具体例が必要

### 削除候補
1. [ノート名] (スコア: 2/10)
   理由: 内容が薄い

## アクション提案
- Memory移動: X件
- Input移動: Y件
- Output移動: Z件
- 追記必要: A件

### `/research [テーマ]` - 知識調査コマンド

**説明**: 指定されたテーマについて網羅的に知識を調査し、詳細なノートを複数作成します。

**重要**: 1項目1ファイルの原則を厳守してください。機能単位、概念単位で必ず別々のファイルを作成します。

**引数**:
- テーマ: 調査したい技術や概念（例: "React Hooks", "TypeScript", "PostgreSQL"）

**実行内容**:
1. 調査項目を洗い出す
   - 基本概念
   - 主要機能（機能ごとに別ファイル）
   - 実装方法
   - ベストプラクティス
   - よくある問題
   - パフォーマンス
   - セキュリティ
   - 関連技術との比較
2. **必須：細分化の徹底**（1つの知識項目 = 1つのファイル）
3. 各ノートの作成（`01_Inbox/`フォルダに）
4. ファイル間にリンクを設定

**使用例**:
```
/research React Hooks
/research TypeScript
/research PostgreSQL
```

**細分化の例**（React Hooksの場合）:
- ✅ 良い例:
  - `YYYYMMDD_React_useStateの使い方.md`
  - `YYYYMMDD_React_useEffectの使い方.md`
  - `YYYYMMDD_React_useContextの使い方.md`
  - `YYYYMMDD_React_Custom Hooksの作成.md`
- ❌ 悪い例:
  - `YYYYMMDD_React Hooksまとめ.md`（全てを1ファイルに）

**注意事項**:
- **最重要**: 1機能1ファイル、1概念1ファイルを厳守
- 表面的な情報ではなく、実践的に使える詳細な内容を書く
- コード例は必ず含める
- 作成したファイルは必ずリンクする

### `/optimize-tag-links [ファイル名]` - タグ・リンク最適化

**説明**: 指定ファイルのタグと双方向リンクを最適化します。

**引数**:
- ファイル名（省略時は現在開いているファイル）

**実行内容**:
1. タグの最適化
   - 現在のタグを分析
   - 内容に基づいて適切なタグを提案
   - 重複・冗長なタグを指摘
   - 階層化の提案
2. 双方向リンクの最適化
   - 現在のリンクを分析
   - 内容から関連ノートを検索
   - 追加すべきリンクを提案
   - 不要・冗長なリンクを指摘
   - リンクの粒度を最適化

**使用例**:
```
/optimize-tag-links
/optimize-tag-links 20241022_API設計.md
```

**タグ階層の推奨**:
- `#memory` - Memory Notes（長期記憶）
- `#input` - Input Notes（短期記憶）
- `#inbox` - Inbox Notes（一時メモ）
- `#meeting` - MTG記録
- `#project/名前` - プロジェクト別
- `#status/draft` - ステータス
- `#technology/分野` - 技術分野別
- `#ai/tools/cursor` - AIツール別（階層化）

**リンクのルール**:
- **リンクすべきもの**: 関連する概念、前提知識、具体例、人物・組織
- **リンク不要なもの**: 一般的な単語、確定していない概念、1回しか出てこない単語
- **粒度のルール**: 1文に1-2個まで、重要な概念のみリンク

## 📊 分析・レビューコマンド

### `/create-dashboards [ダッシュボード名]` - ダッシュボード作成

**説明**: 各種ダッシュボードファイルを作成・更新します。

**引数**:
- ダッシュボード名（省略可）: 作成するダッシュボード
  - `home` または `🏠` - HOMEダッシュボード
  - `projects` または `🎯` - プロジェクトダッシュボード
  - `weekly` または `📊` - 週次ダッシュボード
  - `analytics` または `📈` - 分析ダッシュボード
  - `focus` または `🔥` - アクティブフォーカスダッシュボード
  - `all` - 全てのダッシュボードを作成

**実行内容**:
1. 今日の日付を取得
2. 指定されたダッシュボードの内容を生成
3. `07_System/Dashboards/` フォルダに保存
4. 既存のファイルがある場合は更新

**使用例**:
```
/create-dashboards              # 全てのダッシュボードを作成
/create-dashboards home        # HOMEダッシュボードのみ作成
/create-dashboards 🏠          # HOMEダッシュボードのみ作成
/create-dashboards projects    # プロジェクトダッシュボードのみ作成
/create-dashboards 🎯          # プロジェクトダッシュボードのみ作成
/create-dashboards weekly      # 週次ダッシュボードのみ作成
/create-dashboards 📊          # 週次ダッシュボードのみ作成
/create-dashboards analytics   # 分析ダッシュボードのみ作成
/create-dashboards 📈          # 分析ダッシュボードのみ作成
/create-dashboards focus       # アクティブフォーカスダッシュボードのみ作成
/create-dashboards 🔥          # アクティブフォーカスダッシュボードのみ作成
```

**作成されるダッシュボード**:
- 🏠 HOMEダッシュボード.md - 今日のホームダッシュボード
- 🎯 プロジェクトダッシュボード.md - プロジェクト管理ダッシュボード
- 📊 週次ダッシュボード.md - 週次ダッシュボード
- 📈 分析ダッシュボード.md - 分析ダッシュボード
- 🔥 アクティブフォーカス.md - アクティブフォーカスダッシュボード

**注意事項**:
- DataviewクエリはObsidianのDataviewプラグインが必要です
- 日付は自動的に計算されます
- 既存のファイルがある場合は内容を更新します
- ファイルが存在しない場合は新規作成します
- ファイル名は日本語で作成します

## 🔧 トラブルシューティング

### コマンドが実行されない

**原因**:
- コマンド名のタイプミス
- Cursorの設定問題

**解決策**:
1. コマンド名を正確に入力（`/daily`など）
2. Cursorを再起動
3. `.cursor/commands/`フォルダが正しく配置されているか確認

### ファイルが見つからない

**原因**:
- ファイル名のタイプミス
- ファイルが存在しない

**解決策**:
1. ファイル名を正確に入力
2. ファイルが存在するか確認
3. 引数なしで実行すると、現在開いているファイルが対象になります

### コマンドが期待通りに動作しない

**原因**:
- フォルダ構造が正しくない
- 必要なファイルが存在しない

**解決策**:
1. Brain System Rules（[[AGENTS.md]]）を確認
2. フォルダ構造が正しいか確認
3. 必要なテンプレートファイルが存在するか確認
